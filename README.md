# Прогнозирование популярности музыкальных видео YouTube

Учебный проект по машинному обучению: анализ и моделирование популярности музыкальных видео на YouTube на основе открытого датасета `YouTube-New` (Kaggle).

## Цель проекта

Определить, какие характеристики музыкальных видео влияют на их популярность, и построить модель, которая прогнозирует количество просмотров.

---

## Данные

Использован открытый датасет с Kaggle:

**YouTube Trending Dataset**  
Автор: datasnaek  
Ссылка: https://www.kaggle.com/datasets/datasnaek/youtube-new

В проекте использованы данные стран:
- США (`USvideos.csv`)
- Великобритания (`GBvideos.csv`)
- Россия (`RUvideos.csv`)

Также оставлены только видео категории `Music` (`category_id = 10`).

---

## Этапы выполнения

### 1. Предобработка данных
- Загрузка и объединение датасетов из разных стран  
- Фильтрация музыкальных видео  
- Удаление дубликатов  
- Приведение дат (`publish_time`, `trending_date`)  
- Удаление timezone  
- Обработка пропусков  
- Создание новых признаков:
  - `days_to_trend`
  - `publish_hour`
  - `publish_weekday`
  - `log_views`, `log_likes`, `log_comments`

### 2. Исследовательский анализ данных (EDA)
- Анализ выбросов (`views`, `likes`, `comments`)  
- Гистограммы до и после логарифмирования  
- Корреляционная матрица  
- Динамика популярности по странам  

### 3. Моделирование
Обучены три модели:

| Модель | MAE (log) | R² |
|--------|-----------|------|
| Linear Regression | ~0.63 | ~0.82 |
| Random Forest | ~0.19 | ~0.97 |
| Gradient Boosting | ~0.39 | ~0.93 |

Лучшей моделью оказался **RandomForestRegressor**, объясняющий ~97% вариации просмотров.

### 4. Метрики
Использованы:
- R²  
- MAE (в логарифмах просмотров)

---

## Выводы

- Популярность музыкальных видео наиболее сильно связана с:
  - количеством лайков (`log_likes`)
  - количеством комментариев (`log_comments`)
  - временем попадания в тренды (`days_to_trend`)
- Логарифмирование признаков значительно улучшает качество моделей.
- Модель Random Forest обеспечивает лучшее качество предсказаний.
- Разброс просмотров очень большой (от тысяч до десятков миллионов), поэтому модели лучше работают в лог-пространстве.
